# 服务器配置和环境启动


---

## 1. 机器与任务范围

| 项 | 值 |
| --- | --- |
| GPU | 3 × NVIDIA A800 80GB PCIe |
| 数据盘 | `/root/autodl-tmp`（约 350GB，**重要文件请另备份**） |
| 系统盘 | `/`（约 30GB，勿放大文件） |
| 任务 | EHR Simple SFT（全参、BF16、三卡 DDP）+ constrained-beam 评测 |
| 模型 | 本地 `Qwen3-1.7B` |
| 不包含 | Reasoning activation / RL / VERL / vLLM |

本机必设（已写入 `~/.bashrc`；新终端应自动有）：

```bash
OMP_NUM_THREADS=1
MKL_NUM_THREADS=1
NCCL_P2P_DISABLE=1   # 关 P2P，否则三卡 DDP 可能挂起
```

检查：

```bash
printf 'OMP=%s MKL=%s NCCL_P2P=%s\n' \
  "$OMP_NUM_THREADS" "$MKL_NUM_THREADS" "$NCCL_P2P_DISABLE"
# 期望：1  1  1
```

## 3. 环境：每次新开终端

```bash
source /root/miniconda3/etc/profile.d/conda.sh
conda activate EHR
cd /root/autodl-tmp/project/SIDReasoner_EHR   # 或 cd "$PROJECT_ROOT"

which python   # 应为 .../envs/EHR/bin/python
python -c "import torch; print(torch.__version__, torch.cuda.device_count())"
# 期望：2.6.0+cu124  3
```

登录后常见变量（来自 `~/.bashrc`）：

| 变量 | 含义 |
| --- | --- |
| `PROJECT_ROOT` | 代码根 |
| `MODEL_ROOT` | `/root/autodl-tmp/models/Qwen3-1.7B` |
| `EHR_DATA_DIR` | 默认数据集（可被启动时的 `DATA_DIR` 覆盖） |
| `LOG_ROOT` | `/root/autodl-tmp/logs` |

长期任务请放在 **tmux** 里，避免 SSH 断开中断进程：

```bash
tmux new -s ehr          # 新建
tmux attach -t ehr       # 重连
# Ctrl-b d  ：断开会话（训练继续）
```

## 4. 如何启动训练

参数通过**环境变量**传给 `sft_Qwen3_ehr.sh`，一般不必改 Python 源码。

### 4.1 推荐：先定实验名，再开训

换数据集 / 换配置时，用不同 `EXP_NAME`，避免产物互相覆盖。

```bash
source /root/miniconda3/etc/profile.d/conda.sh
conda activate EHR
cd "$PROJECT_ROOT"

# ---- 按实验修改这几行 ----
export EXP_NAME=mimic3_icd_name_path_0.1_simple_sft
export DATA_DIR=$PROJECT_ROOT/data/EHR/mimic3_icd_name_path_0.1
export DATASET_PREFIX=mimic3_icd          # 对应 index/mimic3_icd.*.json
export OUTPUT_DIR=/root/autodl-tmp/output/$EXP_NAME
export EVAL_RESULT_DIR=/root/autodl-tmp/results/$EXP_NAME
export LOG_FILE=$LOG_ROOT/${EXP_NAME}_train.log
export RUN_NAME=$EXP_NAME
# ----------------------------

mkdir -p "$OUTPUT_DIR" "$EVAL_RESULT_DIR" "$LOG_ROOT"
```

换更大/另一份数据时，例如：

```bash
export EXP_NAME=mimic3_icd_name_path_simple_sft
export DATA_DIR=$PROJECT_ROOT/data/EHR/mimic3_icd_name_path
# DATASET_PREFIX 仍多为 mimic3_icd（以数据目录内文件名为准）
export OUTPUT_DIR=/root/autodl-tmp/output/$EXP_NAME
export EVAL_RESULT_DIR=/root/autodl-tmp/results/$EXP_NAME
export LOG_FILE=$LOG_ROOT/${EXP_NAME}_train.log
export RUN_NAME=$EXP_NAME
```

### 4.2 正式训练命令（三卡全量）

```bash
OMP_NUM_THREADS=1 \
MKL_NUM_THREADS=1 \
NCCL_P2P_DISABLE=1 \
CUDA_VISIBLE_DEVICES=0,1,2 \
NPROC_PER_NODE=3 \
BASE_MODEL="$MODEL_ROOT" \
DATA_DIR="$DATA_DIR" \
DATASET_PREFIX="$DATASET_PREFIX" \
OUTPUT_DIR="$OUTPUT_DIR" \
LOG_DIR="$LOG_ROOT" \
LOG_FILE="$LOG_FILE" \
RUN_NAME="$RUN_NAME" \
WANDB_MODE=offline \
SAMPLE=-1 \
BATCH_SIZE=252 \
MICRO_BATCH_SIZE=2 \
NUM_EPOCHS=10 \
LEARNING_RATE=2e-5 \
CUTOFF_LEN=1024 \
SAVE_TOTAL_LIMIT=3 \
bash sft_Qwen3_ehr.sh
```

详细输出写入 `LOG_FILE`（覆盖写），启动终端几乎看不到 loss；另开窗 `tail -f`（见 §6）。

### 4.3 常用参数

| 变量 | 含义 | 常用值 |
| --- | --- | --- |
| `DATA_DIR` | 数据集顶层目录 | 见上 |
| `DATASET_PREFIX` | index 文件前缀 | `mimic3_icd` |
| `OUTPUT_DIR` | checkpoint / final 落盘根 | `/root/autodl-tmp/output/<EXP>` |
| `SAMPLE` | 抽样；`-1`=全量 | smoke 可用 `128` |
| `BATCH_SIZE` | 有效全局 batch | `252`（须能被 `MICRO×卡数` 整除） |
| `MICRO_BATCH_SIZE` | 每卡 micro | `2`；OOM 改 `1` |
| `NUM_EPOCHS` | epoch 数 | `10` |
| `LEARNING_RATE` | 学习率 | `2e-5` |
| `CUTOFF_LEN` | 序列长度 | `1024` |
| `SAVE_TOTAL_LIMIT` | 保留 `checkpoint-*` 个数 | `3` |
| `RESUME_FROM_CHECKPOINT` | 断点路径 | 空=从头；填某 `checkpoint-*` |

约束：

```text
BATCH_SIZE % (MICRO_BATCH_SIZE × NPROC_PER_NODE) == 0
例：252 ÷ (2 × 3) = 42 次梯度累积
```

### 4.4 小样本 smoke（可选）

```bash
# 仍建议单独 OUTPUT_DIR，例如 .../output/smoke
SAMPLE=128 BATCH_SIZE=30 MICRO_BATCH_SIZE=2 NUM_EPOCHS=1 \
  ...其余同正式训练... \
  bash sft_Qwen3_ehr.sh
```

### 4.5 停止训练

```bash
# 前台：Ctrl-C
pkill -f 'sft_Qwen3_ehr'
pgrep -af 'torchrun|sft_Qwen3_ehr' || echo stopped
```

同一 `OUTPUT_DIR` 不要并行开两个训练。

## 5. 训练落盘产物

以 `OUTPUT_DIR=/root/autodl-tmp/output/<EXP_NAME>` 为例：

```text
$OUTPUT_DIR/
├── manifest/tokenizer_preflight.json
├── checkpoint-<step>/          # 可 resume（含 optimizer 等），最多约 SAVE_TOTAL_LIMIT 个
└── final_checkpoint/           # 训练正常结束后生成；用于评测 / 推理
```

说明：

- `checkpoint-*`：断点续训用  
- `final_checkpoint`：评测用（不要当成 Trainer resume 的完整替代）  
- 日志：`$LOG_FILE`（如 `logs/<EXP>_train.log`）  
- W&B offline：`/root/autodl-tmp/cache/wandb/wandb/offline-run-*`

**同实验从头重跑**：先停进程，再删（或换空）该次 `OUTPUT_DIR`；日志不删也会被覆盖。  
**换数据集**：换新 `EXP_NAME` / 新 `OUTPUT_DIR`，不要直接清空别人的实验目录。

## 6. 如何查看日志与 GPU

```bash
# 训练日志（实时）
tail -f "$LOG_FILE"
# 或
tail -f /root/autodl-tmp/logs/mimic3_icd_name_path_0.1_simple_sft_train.log

# 过滤 loss / eval / 报错
grep -E "'loss'|eval_loss|Error|Traceback|LOAD DATA" "$LOG_FILE" | tail -n 40

# GPU
watch -n 2 nvidia-smi          # -n 2 = 每 2 秒刷新
# 或
nvidia-smi -l 2

# 进程是否还在
pgrep -af 'torchrun|sft_Qwen3_ehr'
```

每个 epoch 结束会打主 `eval_loss`，以及 `title2sid` / `sid2title` 辅助集 loss（辅助集可能重复打印两行，数值相同，属日志重复）。

判断跑完：

```bash
test -f "$OUTPUT_DIR/final_checkpoint/config.json" && echo DONE || echo NOT_YET
tail -n 30 "$LOG_FILE"
```

## 7. 如何评测

评测脚本**单卡**、**非 DDP**。valid 与 test **串行**（共用 `metrics.json`，勿并发）。

训练结束后：

```bash
source /root/miniconda3/etc/profile.d/conda.sh
conda activate EHR
cd "$PROJECT_ROOT"

# 与训练使用同一套 EXP / DATA / OUTPUT
export EXP_NAME=mimic3_icd_name_path_0.1_simple_sft
export DATA_DIR=$PROJECT_ROOT/data/EHR/mimic3_icd_name_path_0.1
export DATASET_PREFIX=mimic3_icd
export OUTPUT_DIR=/root/autodl-tmp/output/$EXP_NAME
export EVAL_RESULT_DIR=/root/autodl-tmp/results/$EXP_NAME
mkdir -p "$EVAL_RESULT_DIR"

# 1) valid
CUDA_VISIBLE_DEVICES=0 python evaluate_Qwen3_ehr.py \
  --base_model="$OUTPUT_DIR/final_checkpoint" \
  --data_dir="$DATA_DIR" \
  --dataset_prefix="$DATASET_PREFIX" \
  --split=valid \
  --result_dir="$EVAL_RESULT_DIR" \
  --batch_size=2 \
  --num_beams=40 \
  --cutoff_len=1024 \
  --sample=-1 \
  2>&1 | tee "$LOG_ROOT/${EXP_NAME}_eval_valid.log"

# 2) test（valid 成功后再跑）
CUDA_VISIBLE_DEVICES=0 python evaluate_Qwen3_ehr.py \
  --base_model="$OUTPUT_DIR/final_checkpoint" \
  --data_dir="$DATA_DIR" \
  --dataset_prefix="$DATASET_PREFIX" \
  --split=test \
  --result_dir="$EVAL_RESULT_DIR" \
  --batch_size=2 \
  --num_beams=40 \
  --cutoff_len=1024 \
  --sample=-1 \
  2>&1 | tee "$LOG_ROOT/${EXP_NAME}_eval_test.log"
```

评测落盘：

```text
$EVAL_RESULT_DIR/
├── evaluation_preflight.json
├── metrics.json
├── evaluation_manifest.json
├── valid_ranklist.jsonl
└── test_ranklist.jsonl
```

查看指标：

```bash
python -m json.tool "$EVAL_RESULT_DIR/metrics.json"
tail -f "$LOG_ROOT/${EXP_NAME}_eval_valid.log"
```

评测 OOM：把 `--batch_size` 改为 `1`，**不要**降低 `--num_beams=40`。

## 8. 最短操作清单

1. SSH 登录 → 建议 `tmux`  
2. `conda activate EHR` → 确认 `OMP/MKL/NCCL_P2P = 1/1/1`  
3. 设定 `EXP_NAME` / `DATA_DIR` / `OUTPUT_DIR` / `LOG_FILE`  
4. 启动 `bash sft_Qwen3_ehr.sh`（带 §4.2 变量）  
5. 另窗 `tail -f` 日志 + `watch -n 2 nvidia-smi`  
6. 出现 `final_checkpoint` 后串行跑 valid → test  
7. 重要结果复制到 `/root/autodl-fs` 或本地下载（数据盘非冗余）

## 9. 联系与约定

- 开新实验请用新的 `EXP_NAME`，勿覆盖他人正在用的 `output/` / `results/`。  
- 改默认全局路径或装包前先沟通，避免弄坏共用 `envs/EHR`。  
- 问题排查优先看：`$LOG_FILE` 是否有 `Traceback`；三卡是否空转挂起（见服务器配置文档 §10.1）。
