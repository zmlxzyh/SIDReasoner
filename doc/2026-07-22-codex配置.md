# 远程服务器 Codex 配置（复用本地代理）

参考：
- [知乎搜索：远程服务器 codex](https://www.zhihu.com/search?type=content&q=%E8%BF%9C%E7%A8%8B%E6%9C%8D%E5%8A%A1%E5%99%A8codex)
- [VSCode Remote SSH 内网服务器使用 Codex](https://zhuanlan.zhihu.com/p/2006479434102899902)

核心思路：远程 Codex 走 `127.0.0.1:7890`，经 SSH `RemoteForward` 打回本机代理，再出网。

以下端口以本机代理实际端口为准，全文 `7890` 统一替换即可。

```text
远程 Codex / VSCode
      ↓
远程 127.0.0.1:7890
      ↓ SSH RemoteForward
本机 127.0.0.1:7890（Clash 等）
      ↓
外网
```



---

## 1. 本机 VSCode Settings

本地 `settings.json`：

```json
{
  "http.proxy": "http://127.0.0.1:7890",
  "http.proxyStrictSSL": false,
  "http.proxySupport": "on"
}
```

## 2. 本机 SSH Config

`~/.ssh/config` 对应 Host 增加：

```sshconfig
Host your-server
    HostName <服务器地址>
    User <用户名>
    RemoteForward 7890 127.0.0.1:7890
```

含义：远程访问 `127.0.0.1:7890` 时，转发到本机代理 `127.0.0.1:7890`。

改完后**断开并重新连接** SSH / Remote，转发才会生效。

## 3. 远程 VSCode Settings

远程侧打开 `Preferences: Open Remote Settings (JSON)`，写入：

```json
{
  "http.useLocalProxyConfiguration": true
}
```

让远程 VSCode 复用本地代理配置。

## 4. 远程 Shell 代理环境变量

在远程 `~/.bashrc`（或 `~/.zshrc`）中添加：

```bash
export http_proxy=http://127.0.0.1:7890
export https_proxy=http://127.0.0.1:7890
export all_proxy=socks5://127.0.0.1:7890
export no_proxy=localhost,127.0.0.1
```

然后：

```bash
source ~/.bashrc
```

## 5. 安装并登录 Codex

代理通后，在远程正常安装 Codex 扩展 / CLI，并完成登录即可。

---

## 检查清单

1. 本机代理已启动，且监听 `7890`
2. SSH 已配置 `RemoteForward`，并已重连
3. 远程 Remote Settings 已开 `http.useLocalProxyConfiguration`
4. 远程 shell 中 `echo $http_proxy` 输出正确
5. 远程可测：`curl -I https://www.google.com`（或任意需代理的外网地址）
