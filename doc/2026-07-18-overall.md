# 生成式 EHR ICD 预测：总体任务与 Prompt 案例

> 更新日期：2026-07-18  
> 本文是项目的 high-level 入口，只说明任务、训练版本和代表性 Task Formats & Prompt Templates。  
> 详细实现见 [`2026-07-18-simpleSFT.md`](./2026-07-18-simpleSFT.md) 与 [`2026-07-18-CoT-RL.md`](./2026-07-18-CoT-RL.md)。

## 1. 基本任务

给定患者当前 visit 之前的 ICD 诊断历史，预测当前 visit 的完整 ICD 疾病集合。

```text
Input:  historical ICD visits
Output: ranked ICD diseases for the next/current target visit
```

每个 ICD 疾病先映射为 Semantic ID（SID）：

```text
ICD9CM:4280 → <a_17><b_93><c_4>
```

训练时，一个多疾病 visit 拆成多个单 SID completion：

```text
same history → SID_A
same history → SID_B
same history → SID_C
```

评测时回到 visit-level，对同一历史生成一个疾病 SID rank-list，再还原为 ICD：

```text
history → constrained beam search → Top-K SID → Top-K ICD
```

主要指标：

- Recall@10/20/30/40。
- oracle-cardinality weighted-F1：每个 visit 取 rank-list 的 Top-|GT| 作为预测集合。

## 2. 输入表示

SID 历史保留 visit 边界和时间顺序：

```text
<visit_start>
<a_8><b_21><c_77>, <a_61><b_2><c_18>
<visit_end>
<visit_start>
<a_14><b_62><c_9>
<visit_end>
```

第一版只使用历史 ICD，不使用 procedures、drugs 或时间。具体疾病实体在 SID 任务和 reasoning 中使用完整 SID 指代。

# Part I · Simple SFT

Simple SFT 不使用富语料、CoT 或 RL，只训练基础 SID–疾病语义对齐和生成式 ICD 预测。

## 3. Simple SFT Tasks

| Case | Task | Input → Output | Purpose |
|:---:|---|---|---|
| 1 | Disease identification | Disease text → SID | 将疾病语义映射到 SID |
| 2 | Disease identification | SID → Disease text | 防止 SID 退化为纯编号 |
| 3 | Next-diagnosis prediction | SID history → Disease text | 从 SID 病史恢复疾病语义 |
| 4 | Next-diagnosis prediction | Text history → Disease text | 纯文本疾病预测对照 |
| 5 | Next-diagnosis prediction | SID history → SID | **最终主任务** |
| 6 | Next-diagnosis prediction | Text history → SID | 文本病史与 SID 输出对齐 |

## 4. Simple SFT · Task Formats & Prompt Templates

### Case 1 · Disease Text → SID

> **System** ▸ You map ICD disease descriptions to semantic identifiers.  
> **User** ▸ Which disease has the description "Congestive heart failure, unspecified"?  
> **Assistant** ▸ `<a_17><b_93><c_4>`

### Case 2 · SID → Disease Text

> **System** ▸ You map semantic identifiers to ICD disease descriptions.  
> **User** ▸ What disease does `<a_17><b_93><c_4>` represent?  
> **Assistant** ▸ Congestive heart failure, unspecified.

### Case 3 · SID History → Disease Text

> **System** ▸ Predict one possible diagnosis in the next visit from the chronological SID history.  
> **User** ▸ The patient's chronological diagnosis history is:  
> `<visit_start> <a_8><b_21><c_77> <visit_end>`  
> `<visit_start> <a_14><b_62><c_9> <visit_end>`  
> Return one possible disease description for the next visit.  
> **Assistant** ▸ Congestive heart failure, unspecified.

### Case 4 · Text History → Disease Text

> **System** ▸ Predict one possible diagnosis in the next visit from the chronological diagnosis history.  
> **User** ▸ The patient's chronological diagnosis history is:  
> Visit 1: Essential hypertension, unspecified.  
> Visit 2: Coronary atherosclerosis of native coronary artery.  
> Return one possible disease description for the next visit.  
> **Assistant** ▸ Congestive heart failure, unspecified.

### Case 5 · SID History → SID

> **System** ▸ Predict one possible diagnosis SID in the patient's next visit.  
> **User** ▸ The patient's chronological diagnosis history is:  
> `<visit_start> <a_8><b_21><c_77> <visit_end>`  
> `<visit_start> <a_14><b_62><c_9> <visit_end>`  
> Return exactly one disease SID.  
> **Assistant** ▸ `<a_17><b_93><c_4>`

### Case 6 · Text History → SID

> **System** ▸ Predict one possible diagnosis SID from the chronological diagnosis history.  
> **User** ▸ The patient's chronological diagnosis history is:  
> Visit 1: Essential hypertension, unspecified.  
> Visit 2: Coronary atherosclerosis of native coronary artery.  
> Return exactly one disease SID.  
> **Assistant** ▸ `<a_17><b_93><c_4>`

Simple SFT 详细的数据格式、训练和评测流程见 [`2026-07-18-simpleSFT.md`](./2026-07-18-simpleSFT.md)。

# Part II · Rich Corpus, CoT & RL

该版本在 Simple SFT 上增加疾病级/患者级富语料、CoT 推理激活和 VERL-GRPO。

## 5. Reasoning & RL Tasks

| Stage | Case | Task | Input → Output | Purpose |
|---|:---:|---|---|---|
| Offline Corpus Distillation | 7A | Disease semantic analysis | Disease metadata → Structured semantic card | 生成有来源的结构化医学语义 |
| Offline Corpus Distillation | 7B | Disease SID integration | Semantic card + SID → SID narrative | 将疾病语义绑定到 SID |
| Offline Corpus Distillation | 8A | Patient reasoning generation | History + hidden reference GT → `reasoning_path` | 生成不泄漏答案的患者级推理 |
| Offline Corpus Distillation | 8B | Patient narrative integration | History + reasoning + reference GT → narrative | 构造 Stage 1 患者富语料 |
| Stage 1 | 7/8 | Rich SID language modeling | SID narrative → next-token LM | 学习疾病与病程富语义 |
| Stage 2 | 9 | Reasoning activation | SID history → CoT + target SID | 激活“先推理、再预测”行为 |
| Stage 3 | 10 | Outcome-only GRPO | SID history → free rollout | 用最终疾病命中优化策略 |

## 6. Rich Corpus · Task Formats & Prompt Templates

### Case 7A · Structured Disease Semantic Analysis

> **System** ▸ You organize traceable ICD disease metadata into a structured semantic card. Do not add unsupported medical facts.  
> **User** ▸ Analyze the following disease metadata:  
> Disease name: `{disease_name}`  
> Parent category: `{parent_name}`  
> Ontology path: `{ontology_path}`  
> Sourced knowledge: `{knowledge_with_source_ids}`  
> Return JSON with `definition`, `body_system`, `broader_category`, `clinical_context`, `related_concepts`, and `source_ids`. Use empty values when evidence is unavailable.  
> **Assistant** ▸ `{"definition":"...","body_system":"...","source_ids":["..."]}`

### Case 7B · Disease SID Narrative Integration

> **System** ▸ Convert structured disease semantics into a coherent SID-centered narrative.  
> **User** ▸ Disease SID: `{sid}`  
> Structured semantic card: `{semantic_card}`  
> Whenever you refer to the specific disease, use `{sid}` exactly. Do not use its disease name, ICD code, or aliases. Do not add unsupported medical facts.  
> **Assistant** ▸ `{sid}` belongs to ... . In this context, `{sid}` is associated with ... .

### Case 8A · Reference-guided Patient Reasoning

教师可以看到 GT 作为内部 guidance，但输出不能泄漏非历史 GT。

> **System** ▸ You are a longitudinal EHR prediction analyst. Reason as if you only know the history.  
> **User** ▸ Historical visits: `{history_with_sids_and_descriptions}`  
> Reference next-visit diseases for internal guidance only: `{reference_gt}`  
> Output only a concise first-person reasoning monologue. Never mention or imply knowledge of the reference diseases. When referring to a disease, use only a SID already present in the history. Do not predict a specific next disease. End with a non-deterministic summary.  
> **Assistant** ▸ I observe that `<a_...><b_...><c_...>` persists across visits ... related conditions may remain relevant in a subsequent visit.

### Case 8B · Patient Narrative Integration

> **System** ▸ Integrate the longitudinal history and reasoning into one coherent SID-centered narrative.  
> **User** ▸ Historical SID visits: `{history_sid_visits}`  
> Reasoning path: `{reasoning_path}`  
> Reference next-visit SID set: `{reference_sids}`  
> Start by preserving the complete chronological visit sequence. Refer to specific diseases only by SID. The outcome section may connect the history to the reference SID set. Do not introduce any other SID.  
> **Assistant** ▸ `<visit_start> ... <visit_end>` ... the subsequent visit includes `{reference_sids}`.

`reasoning_path` 用于 Stage 2，不能包含非历史 GT；`integrated_narrative` 只用于 Stage 1，可以在结果部分包含 reference GT。

## 7. CoT & RL · Task Formats & Prompt Templates

### Stage 1 Consumption · Rich SID Language Modeling

疾病级和患者级富语料作为纯文本训练：

```text
Input:  SID-centered disease or patient narrative
Label:  the same token sequence (next-token language modeling)
```

不要求模型在 Stage 1 输出 `<think>`。

### Case 9 · CoT Reasoning Activation

> **System** ▸ You analyze longitudinal diagnosis patterns represented by semantic IDs. Reason only from the provided history. Do not claim certainty or clinical causality. After reasoning, output exactly one disease SID as a possible diagnosis in the next visit.  
> **User** ▸ The patient's chronological diagnosis history is:  
> `<visit_start> ... <visit_end>`  
> `<visit_start> ... <visit_end>`  
> Think step by step, then return one possible disease SID.  
> **Assistant** ▸ `<think>`  
> I observe that `<a_...><b_...><c_...>` persists across visits, while another historical SID appears more recently. Related conditions may remain relevant in the next visit.  
> `</think>`  
> `<a_x><b_y><c_z>`

一个 visit 只蒸馏一条 reasoning，再与该 visit 的每个真实目标 SID 分别配对。

### Case 10 · VERL-GRPO Free Rollout

RL prompt 与 Case 9 同构，但不提供 teacher reasoning：

```text
Prompt:
  historical SID visits

Free rollout:
  <think>
  model-generated reasoning
  </think>
  <a_x><b_y><c_z>
```

第一版奖励只有最终结果：

```text
reward = 1.0  if answer_sid ∈ complete next-visit GT SID set
reward = 0.0  otherwise
```

RL rollout 不挂 SID Trie，也不使用格式奖励、reasoning 奖励或 SID prefix 部分奖励。最终评测时才对答案 SID 使用合法 trie。

## 8. Training and Evaluation Summary

| Version | Training | Inference |
|---|---|---|
| Simple SFT | Cases 1–6 | Direct constrained beam → SID rank-list |
| Stage 1 | Cases 1–8（可选通用/医学推理保持任务） | SID–language alignment checkpoint |
| Stage 2 | Case 9 + Simple replay | Free reasoning + one SID |
| Stage 3 | Case 10, VERL-GRPO | Outcome-optimized reasoning policy |
| Final evaluation | — | Free reasoning once → fixed reasoning + constrained SID beam |

CoT/RL 版本的富语料文件、Stage 1/2/3 数据粒度和 VERL 配置见 [`2026-07-18-CoT-RL.md`](./2026-07-18-CoT-RL.md)。

## 9. 关键原则

- 训练可按目标疾病拆分，评测和 RL 必须回到完整 visit。
- reasoning 中引用的具体疾病 SID 必须来自历史。
- 富语料教师可见 GT 仅用于 internal guidance，不能泄漏到 `reasoning_path`。
- RL rollout 自由生成；合法 SID 硬约束只用于最终 rank-list 评测。
- reasoning 是预测依据的语言化表达，不代表真实临床因果或诊断建议。
- 所有实验最终以 Recall@K、weighted-F1 和泄漏审计共同判断。
